================================================================================
  CCRT Module — Validation Strategy Options (n=400)
================================================================================

현재 상태: Train 280 (70%) / Val 40 (10%) / Test 80 (20%)
문제점: Val 40명(양성 12명)으로 early stopping 불안정, Test 80명으로 커브 계단형


────────────────────────────────────────────────────────────────────────────────
Option A: 5-Fold Stratified CV (OOF)
────────────────────────────────────────────────────────────────────────────────

방법:
  전체 400명 → 5-Fold Stratified CV → OOF(Out-of-Fold) 예측값으로 모든 평가

구조:
  Fold 1: Train 320명 → Test 80명 (OOF)
  Fold 2: Train 320명 → Test 80명 (OOF)
  Fold 3: Train 320명 → Test 80명 (OOF)
  Fold 4: Train 320명 → Test 80명 (OOF)
  Fold 5: Train 320명 → Test 80명 (OOF)
  → 400명 전체에 대한 예측값 확보

장점:
  - 400명 전부 학습에 활용
  - OOF 400명으로 커브 부드러움
  - n < 1000 논문에서 가장 표준적 방식
  - 별도 val set 불필요 (early stopping은 fold 내부에서 처리)

단점:
  - "완전히 unseen" test set 없음
  - 하이퍼파라미터 튜닝 시 정보 누수 가능성 (Nested CV로 해결)

적합한 상황:
  - 단일 기관, 외부 검증 없을 때
  - 논문 1차 제출

참고 논문 사례:
  - 대부분의 clinical prediction model 개발 논문 (TRIPOD Statement)


────────────────────────────────────────────────────────────────────────────────
Option B: 80/20 Train-Test Split + 5-Fold CV on Train
────────────────────────────────────────────────────────────────────────────────

방법:
  Test 80명 holdout → Train 320명에서 5-Fold CV → 최종 모델로 Test 평가

구조:
  [Train 320명] ──→ 5-Fold CV (모델 선택/튜닝)
                 ──→ 전체 320명으로 최종 학습
  [Test 80명]   ──→ 최종 평가 (1회)

장점:
  - Holdout test set 있음 → 리뷰어 설득 쉬움
  - 학습/평가 완전 분리

단점:
  - Test 80명(양성 24명) → 성능 지표 분산 큼
  - 커브 계단형 (해결: OOF로 보조 제시)
  - 분할 seed에 따라 결과 변동

적합한 상황:
  - 데이터가 더 클 때 (n > 1000)
  - 외부 검증 데이터가 있을 때 (이 방식 + 외부 검증)


────────────────────────────────────────────────────────────────────────────────
Option C: Nested CV (이중 교차 검증)
────────────────────────────────────────────────────────────────────────────────

방법:
  Outer 5-Fold (평가용) × Inner 5-Fold (튜닝용) = 총 25회 학습

구조:
  Outer Fold 1: [Test 80명]
    Inner Fold 1-1 ~ 1-5: 하이퍼파라미터 최적화
    → 최적 파라미터로 Outer Train 학습 → Outer Test 평가
  ... (5회 반복)

장점:
  - 가장 편향 없는 성능 추정
  - 하이퍼파라미터 튜닝까지 완전 격리
  - 통계적으로 가장 엄밀

단점:
  - 계산량 25배
  - 구현 복잡
  - 400명에서는 과한 감이 있음

적합한 상황:
  - 하이퍼파라미터를 적극적으로 튜닝하는 경우
  - 모델 간 비교의 통계적 엄밀성이 필요한 경우
  - 리뷰어가 과적합을 강하게 의심하는 경우


────────────────────────────────────────────────────────────────────────────────
Option D: Repeated K-Fold CV
────────────────────────────────────────────────────────────────────────────────

방법:
  5-Fold CV를 서로 다른 seed로 10회 반복 → 총 50회 fold 평가

구조:
  Repeat 1 (seed=42):  5-Fold CV → AUC 5개
  Repeat 2 (seed=123): 5-Fold CV → AUC 5개
  ...
  Repeat 10 (seed=999): 5-Fold CV → AUC 5개
  → AUC 50개의 평균 ± SD

장점:
  - 분할 운에 의한 편향 최소화
  - 성능 분산 추정이 가장 정확
  - Option A와 결합 가능

단점:
  - 계산량 10배
  - 각 반복의 OOF가 다름 (앙상블 가능)

적합한 상황:
  - Option A 결과의 robustness 확인
  - 모델 간 차이가 미미해서 통계적 검정이 필요할 때


────────────────────────────────────────────────────────────────────────────────
Option E: Stratified Group K-Fold
────────────────────────────────────────────────────────────────────────────────

방법:
  같은 그룹(기관/시기/주치의)의 환자가 같은 fold에 배정

구조:
  기관 A 환자 → Fold 1, 2에만 배정
  기관 B 환자 → Fold 3, 4에만 배정
  기관 C 환자 → Fold 5에만 배정

장점:
  - 기관 간 데이터 누수 완전 방지
  - 다기관 연구에서 필수

단점:
  - 그룹 정보 필요
  - 그룹 크기 불균형 시 fold 크기 불균형

적합한 상황:
  - 다기관 데이터
  - 같은 환자의 여러 에피소드가 있을 때


────────────────────────────────────────────────────────────────────────────────
Option F: Leave-One-Out CV (LOOCV)
────────────────────────────────────────────────────────────────────────────────

방법:
  399명 학습 → 1명 테스트, 400회 반복

장점:
  - 최대한 많은 학습 데이터 활용
  - 편향 최소

단점:
  - 계산량 400배
  - 분산 높음 (fold 간 학습 데이터 99.75% 중복)
  - 요즘 거의 사용하지 않음

적합한 상황:
  - n < 50 극소 데이터
  - 현재 400명에서는 비권장


────────────────────────────────────────────────────────────────────────────────
Option G: Bootstrap .632+ 평가
────────────────────────────────────────────────────────────────────────────────

방법:
  부트스트랩 샘플(복원추출 400명)로 학습
  → OOB(Out-of-Bag, ~37% 미포함 샘플)로 평가
  → 200회 반복 후 .632+ 보정

공식:
  Error_632+ = (1-w) * Error_train + w * Error_OOB
  w = 0.632 / (1 - 0.368 * R)  (R: overfitting rate)

장점:
  - 작은 데이터에서 편향-분산 균형 최고
  - 이론적으로 가장 정확한 성능 추정

단점:
  - 구현 복잡
  - 리뷰어가 익숙하지 않을 수 있음
  - OOB 샘플 크기가 매번 다름

적합한 상황:
  - 통계적 엄밀성이 최우선일 때
  - 방법론 논문


────────────────────────────────────────────────────────────────────────────────
Option H: Temporal Split (시간 기반 분할)
────────────────────────────────────────────────────────────────────────────────

방법:
  과거 환자 → Train, 최근 환자 → Test

구조:
  2019~2022 환자 (300명) → Train + 5-Fold CV
  2023~2024 환자 (100명) → Temporal Test

장점:
  - 실제 임상 적용과 동일한 조건 (미래 환자 예측)
  - 외부 검증 없을 때 가장 설득력 있음
  - 시간에 따른 데이터 drift 감지 가능

단점:
  - 시기별 환자 특성/치료 패턴 차이 존재 가능
  - 최근 환자가 적으면 test set 부족

적합한 상황:
  - 단일 기관에서 외부 검증 불가할 때
  - 치료 프로토콜이 시간에 따라 변했을 때
  - 리뷰어가 "prospective하게 적용 가능한가?" 물을 때
